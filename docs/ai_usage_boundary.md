# AI 使用边界说明

## 1. 允许 AI 参与的范围

* 静态页面接线与前端脚本改造
* FastAPI 接口、模型、迁移、脚本和测试生成
* Nginx / Docker Compose 编排文件编写
* 文档初稿整理

## 2. 不允许 AI 直接替代的判断

* 不允许跳过自动化测试就宣称功能完成
* 不允许在没有迁移的情况下直接改数据库结构
* 不允许臆造设计文档中不存在的业务规则
* 不允许把前台和后台鉴权链路混为一体
* 不允许绕开 `memory-bank/implementation_plan.md` 的顺序直接跨步开发

## 3. AI 产出必须满足的验证要求

* 每个关键步骤都要有可执行验证
* 后端改动至少补对应单元、接口或集成测试
* 主链路改动要补页面级或 e2e 回归
* 部署改动要实际运行 `docker compose` 验证

## 4. 人工复核重点

* 演示账号、端口、部署入口是否正确
* 迁移脚本是否兼容 PostgreSQL 真实运行环境
* 推荐、积分、订单等答辩重点链路是否真的在页面层更新
* 上传、鉴权、限流等安全边界是否仍然生效

## 5. 当前项目的实际落地原则

* AI 可以负责“实现 + 测试 + 文档”的大部分重复性工作。
* 但所有可演示结论必须以仓库内真实测试结果和真实运行结果为准，而不是以模型口头描述为准。
